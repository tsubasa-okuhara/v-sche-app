## 主な特徴

### ✅ 電子帳簿保存法完全対応

| 要件 | 実装 |
|------|------|
| DPI 200dpi以上 | 自動検証 |
| カラー画像（RGB） | 必須チェック |
| SHA-256ハッシュ値 | 自動計算 |
| タイムスタンプ記録 | 全操作に自動付与 |
| 監査ログ | 全操作を自動記録 |
| 論理削除 | 完全削除禁止 |
| 変更前後の値記録 | 自動記録 |
| 変更者記録 | 自動記録 |
| 変更理由記録 | 必須入力 |

### 🚀 すぐに使える

- **外部サービス不要**: SQLite + ローカル画像保存
- **モック OCR 搭載**: デモ可能、Google Vision API対応可能
- **自動初期化**: 初回実行時に DB・フォルダ自動作成
- **日本語 UI**: 完全日本語対応

### 🔒 セキュリティ

- 物理削除禁止（論理削除のみ）
- 全操作が監査ログに記録
- 画像の原本性を SHA-256 で確保
- 変更履歴により改ざん検知可能

## 4ページの機能

### Page 1: 📝 レシート登録
1. 画像をアップロード（ドラッグ&ドロップ対応）
2. DPI・色モード自動検証
3. OCR自動抽出
4. フォーム自動入力
5. 情報確認・修正
6. 登録ボタンで保存

### Page 2: 🔍 検索・一覧
1. 検索条件を指定（日付・金額・取引先）
2. 検索ボタン実行
3. 結果をテーブル表示
4. 統計表示（合計・件数・平均）

### Page 3: 📋 詳細・編集
1. レシートを選択
2. 画像を大表示
3. 全情報を確認
4. 必要な項目を編集
5. **変更理由を入力（必須）**
6. 保存 or 削除

### Page 4: 📊 監査ログ
1. 全操作を時系列表示
2. レシートID でフィルタ
3. 操作タイプ（INSERT/UPDATE/DELETE）でフィルタ
4. 変更前後の値を確認

## よくある質問

**Q1: 何をインストールすればいい？**  
A: `pip install -r requirements.txt` だけで OK。依存パッケージは 4 個のみ。

**Q2: Google Vision API は必須？**  
A: いいえ。デフォルトはモック（デモ）OCR。任意で設定可能。

**Q3: データベースをリセットしたい**  
A: `receipt_database.db` を削除してアプリ再起動。

**Q4: 画像はどこに保存される？**  
A: `images/YYYY/MM/` フォルダに自動保存。相対パス管理。

**Q5: 削除したレシートは復元できる？**  
A: 論理削除なので、DB管理者が手動復元可能。

**Q6: 複数ユーザー対応？**  
A: 本バージョンは単一ユーザー。拡張については ARCHITECTURE.md 参照。

## トラブルシューティング

### エラー: ModuleNotFoundError
```bash
pip install -r requirements.txt
```

### エラー: 画像が保存できない
```bash

```

### エラー: データベースが壊れた
```bash
rm receipt_database.db  # 削除して再初期化
streamlit run app.py
```
